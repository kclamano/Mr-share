# Contributing to MRShare

Thank you for your interest in contributing to MRShare.

We welcome developers, designers, students, and open-source enthusiasts of all skill levels.

---

## Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub.

### 2. Clone Your Fork

```bash
git clone https://github.com/sajansonkar-cmd/Mr-share
cd mrshare
```

### 3. Install Dependencies

```bash
npm install
```

### 4. Start Development Server

```bash
node app.js
```

---

## Pull Request Process

1. Create a new branch.
2. Make your changes.
3. Test your changes locally.
4. Commit using meaningful messages.
5. Push your branch.
6. Open a Pull Request.

---

## Good First Issues

New contributors can work on:

* UI improvements
* Responsive design fixes
* Error page redesign
* Documentation updates
* Code cleanup
* Accessibility improvements

---

## Intermediate Tasks

* QR code generation

## Advanced Tasks

* File encryption

* Cloud storage integration

---

## Code Style

* Use meaningful variable names.
* Keep functions small and focused.
* Add comments where necessary.
* Avoid unnecessary dependencies.
* Follow existing project structure.

---

## Reporting Bugs

Please include:

* Operating system
* Browser (if applicable)
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots (if possible)

---

## Feature Requests

Feature requests should include:

* Problem being solved
* Proposed solution
* Expected impact
* Possible implementation ideas

---

## Community Guidelines

Be respectful and professional.

We aim to maintain a welcoming and inclusive environment for all contributors.
